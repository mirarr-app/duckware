Frame

FFmpeg's full power behind a native desktop UI — no Electron, no guesswork.

View Source

Desktop Interface

preview.png

What is Frame?

FFmpeg is the undisputed engine for media conversion, but its CLI surface area is enormous — hundreds of flags across video codecs, audio streams, image sequences, subtitle tracks, and metadata containers. Most GUI wrappers either hide that complexity behind useless presets or ship as sluggish Electron shells that wrap a subprocess call.

Frame takes a different approach: a native desktop app written in Rust with a GPUI-CE front end, backed by a reusable conversion core that handles source probing, argument generation, compatibility validation, task control, and progress parsing. You get granular control over every FFmpeg parameter without memorizing flag syntax.

Architectural Highlights & Strengths

  • Reusable Rust Conversion Core

    The FFmpeg logic lives in a dedicated Rust library separate from the UI layer. Source files are probed for stream compatibility before conversion starts, arguments are generated programmatically, and progress is parsed from FFmpeg's output in real time. The core can be reused or extended without touching the interface code.

  • Granular Stream Control

    Frame exposes settings for video, audio, image, subtitle, and metadata streams individually — not just container format and bitrate sliders. Transcode a video while remapping audio channels, burn in subtitles, strip metadata, or convert an image sequence, all from the same interface with validation before the job runs.

  • Native GPUI Performance

    Built on GPUI-CE rather than a web view, Frame renders as a true native desktop application. That means lower memory footprint, responsive UI during long encodes, and no Chromium dependency bundled into your media toolchain.
