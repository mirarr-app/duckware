Audit · What it is

FFmpeg’s power, without memorizing the flag encyclopedia.

FFmpeg remains the default engine for serious media conversion, but its CLI surface is vast — codecs, filters, stream maps, pixel formats, subtitle burn-in, metadata surgery. Most wrappers either bury that depth under a few presets or ship as Electron shells that merely decorate a subprocess call.

Frame takes the opposite bet: a native Rust desktop app with a GPUI-CE front end, and a dedicated conversion library (frame-core) that owns probing, argument generation, compatibility checks, queue control, and live progress parsing. You dial video, audio, image, subtitle, and metadata streams independently — then watch the encode update while it runs.

Why it stands out

A conversion core first, a UI second.

01 — CORE

Reusable Rust pipeline

Source inspection via ffprobe, a JSON compatibility matrix, programmatic FFmpeg argument building, and real-time stderr progress parsing live in frame-core — separate from the GPUI shell, ready to extend without rewriting the UI.

02 — STREAMS

Per-stream control

Video encoders from libx264/x265 and SVT-AV1 to ProRes and platform hardware paths; audio codecs and filters; subtitle selection or burn-in; metadata preserve/clean/replace — validated before the job starts.

03 — NATIVE

GPUI, not a web view

No Chromium tax. Frame renders as a true desktop application with a custom titlebar, workspace and logs views, concurrent encode queueing, and bundled FFmpeg/FFprobe binaries that advertise encoder capability at startup.

Get it

Signed-manifest updates, installers on every major platform.

Quick install paths

Prebuilt binaries ship for macOS, Windows, and Linux. Release builds include a signed-manifest updater that verifies Ed25519 signatures and SHA-256 hashes before installing through a bundled helper.

$ winget install --id 66HEX.Frame -e

$ brew tap 66HEX/frame && brew install --cask frame

$ flatpak install flathub io.github._66HEX.Frame
