Audit · Showcase #12

Maps that compose like UI, not like SDKs

React teams keep rediscovering the same tax: MapLibre (or Mapbox) is capable, but wiring styles, markers, popups, and chrome into a polished product UI is a project of its own. mapcn treats that surface as a component problem — the same copy-into-your-repo model popularized by shadcn/ui, powered by MapLibre GL and Tailwind.

You do not import a sealed black-box package. The CLI drops source into your tree so you can restyle pins, swap tile providers, and compose routes the way you already compose buttons and dialogs. Theme awareness is built in: basemaps and chrome can follow light and dark without a second integration path.

React MapLibre shadcn/ui

Why it lands

Three strengths that matter in practice

01

Own the source

Installation via the shadcn CLI copies map primitives into your project. Debugging a marker quirk or restyling a popup is a local edit — not a fight with opaque props buried in node_modules.

02

Declarative map UI

Markers, popups, tooltips, routes, and controls compose as React children. You get MapLibre’s rendering depth without abandoning the mental model of a modern component library.

03

Theme-native defaults

CARTO basemap tiles flip with light/dark themes out of the box. Pair that with Tailwind tokens already in a shadcn stack and the map stops looking like a foreign iframe.

Kit

What you actually ship

Beyond a bare canvas, mapcn ships the pieces product maps usually need on day one — and docs cover clusters, arcs, GeoJSON layers, and custom styles when you outgrow the basics.

Markers & popups

Rich pins with labels, tooltips, and popups — including draggable markers when interaction is the point.

Routes & paths

Draw polylines and route overlays for directions, deliveries, or any path narrative on the map.

Map controls

Zoom, compass, locate, and fullscreen chrome that matches shadcn-adjacent UI density.

MapLibre depth

Full engine access when you need custom layers, clustering, arcs, or alternate tile providers.
