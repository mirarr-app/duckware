mapcn

Drop-in map components for React — theme-aware, shadcn-native, powered by MapLibre GL.

What is mapcn?

Adding maps to a React app usually means wrestling with MapLibre or Mapbox boilerplate, custom CSS that fights your design system, and a pile of glue code for markers, popups, and controls. mapcn packages all of that into copy-paste components that follow the same conventions as shadcn/ui.

Built on MapLibre GL and styled with Tailwind, the library ships ready-to-use pieces for basemaps, markers, popups, route drawing, and navigation controls. Install with a single CLI command, drop a component into your page, and you get a polished map that respects light and dark mode without extra configuration.

Architectural Highlights & Strengths

  • shadcn/ui-Native Patterns

    Components are designed to slot into existing shadcn projects — same Tailwind tokens, same compositional style, same copy-via-CLI workflow. You are not adopting a separate design language; the map UI inherits your app's visual system and theme switching automatically.

  • Composable Map Primitives

    Rather than one monolithic map widget, mapcn exposes building blocks — basemap containers, marker layers, popups, tooltips, route overlays, and zoom or locate controls. Compose them declaratively in JSX to build dashboards, store locators, or logistics views without touching low-level MapLibre APIs directly.

  • Zero-Config Defaults, Full MapLibre Escape Hatch

    Sensible basemap and control defaults get you rendering immediately via CARTO tiles, with the option to swap in OpenStreetMap, MapTiler, Stadia, or any other MapLibre-compatible provider. When you need raw engine access, the full MapLibre GL API remains available underneath.

Quick Setup

Add components to an existing shadcn project through the mapcn CLI — no manual wiring of stylesheets or provider tokens required:

terminal
pnpm dlx shadcn@latest add @mapcn/map
✓ maplibre-gl installed · map component added

# Then import and render in your React app
import { Map, MapControls } from "@/components/ui/map"
