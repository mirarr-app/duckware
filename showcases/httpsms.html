httpSMS

Your Android phone, exposed as a programmable SMS gateway over HTTP.

Visit httpSMS View Source

Platform Overview

httpsms-header.png

What is httpSMS?

Commercial SMS APIs work well in the US and Europe, but many regions cannot buy virtual phone numbers at all. Developers in those markets are left stitching together brittle workarounds or paying for gateways that do not support their local carriers. httpSMS takes a different approach: your own Android handset becomes the gateway.

Install the Android app, pair it with the service, and send SMS messages by calling a straightforward HTTP API. Inbound texts arriving on the phone are forwarded to a webhook you configure. The entire stack — API server, web dashboard, and mobile client — is open source and self-hostable.

Architectural Highlights & Strengths

  • Real SIM, Real Delivery

    Messages go out through your phone's native SMS stack, not a rented short code. That means local carrier rates, local number identity, and delivery to numbers that commercial APIs often cannot reach. Your Android device stays online and the API queues outbound messages until the app picks them up via push notification.

  • Bidirectional HTTP Integration

    Outbound is a single REST call — POST your message body, from number, and recipient. Inbound SMS events are delivered to your webhook endpoint as they arrive on the handset, so you can build two-way flows like OTP verification, appointment reminders, or alert systems without polling.

  • Full-Stack Open Source

    The monorepo ships three components: a Go API backend, an Android companion app, and a Nuxt/Vuetify web dashboard for managing phones, API keys, and message history. Docker Compose brings the server side up locally, and the hosted version at httpsms.com is available if you prefer not to run your own infrastructure.

Sending a Message

The API accepts a JSON payload and returns immediately while the Android app handles delivery asynchronously:

curl
curl -X POST https://api.httpsms.com/v1/messages/send \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello!", "from": "+1234567890", "to": "+0987654321"}'

202 Accepted — message queued for delivery

Self-Hosting

Run the full stack on your own infrastructure with Docker Compose. Configure Firebase credentials for push notifications and you have a private SMS gateway:

bash
git clone https://github.com/NdoleStudio/httpsms.git
cp web/.env.docker web/.env
cp api/.env.docker api/.env
docker compose up --build
