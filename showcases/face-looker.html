Face Looker

One portrait becomes a gaze atlas — then a React face that locks onto the cursor.

View source Run on Replicate Live demo

What it is

A two-act pipeline for eyes that track

Face Looker turns a centered 512×512 portrait into a dense grid of gaze directions via Replicate’s expression tools, then ships a React hook that picks the right frame from pointer position. The result feels like live eye contact — without a webcam, WebGL shader, or 3D head model.

01

Generate the atlas

Upload once to the dedicated Replicate model — or drive main.py with step/min/max for denser grids. Default is an 11×11 set (121 frames); tighten the step for smoother arcs.

02

Ship the frames

You get numbered WebPs, a sprite sheet, a CSV index, and even a vanilla HTML/JS zip from the hosted model. Drop the faces into public/ and you’re halfway to interactive.

03

Wire the hook

useGazeTracking maps container-relative pointer (and touch) into the nearest grid cell, swapping images at interactive rates. Match P_MIN / P_MAX / STEP to whatever you generated.

Metaphor: 11×11 gaze cells · active sample at center

Why it lands

Strengths worth stealing

One-click atlas

The Replicate face-looker model collapses GPU wrangling into upload → zip. Sprite sheet, frames, and starter HTML arrive together — ideal when you want the effect before you want the pipeline.

Hook, not a black box

Beyond the FaceTracker component, useGazeTracking exposes the mapping so you can mask circles, stack multiple faces, or drive custom layouts without forking the math.

Touch is first-class

Finger motion drives the same grid on mobile. Portfolio headers and avatar widgets keep their “watching you” trick past the mouse — rare for effects that start as desktop demos.

In the wild

Drop-in React surface

Copy faces into public, drop the hook and component into src, and you’ve got a header that looks back. Density is a dial: larger step values mean fewer images and cheaper generation; smaller steps buy smoother saccades.

// Pointer → nearest gaze_px{X}_py{Y}_256.webp
