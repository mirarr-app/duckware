DuckWare audit

A tray app that earns its keep by staying out of the way.

Everyone’s Downloads folder becomes a landfill: installers next to invoices, screenshots mixed with zips, Takeout dumps waiting for a weekend that never comes. Cloud “organizers” want to read filenames or sync metadata off-machine — a trade most privacy-minded users refuse.

Mouzi is a free, MIT-licensed Windows and Linux utility built with Tauri, Rust, and React. It lives in the system tray, watches the folders you choose, and moves new files into tidy destinations using a first-match-wins rules engine — extensions, regex, path placeholders like {year}/{month}, plus .mouziignore when you need to leave something alone. Silent, manual, or paused modes per folder; local SQLite history with one-click undo; optional Google Takeout archive import that extracts and sorts in one pass. Roughly ~5 MB RAM, no cloud, no telemetry by default.

Rules engine

First match wins. Chaos becomes shelves.

Sensible defaults cover images, documents, archives, installers, and media. Then you extend with regex, priority order, and dynamic destination paths.

If extension
jpg · png · webp · gif
→ Downloads/Images/
If extension
pdf · docx · xlsx
→ Documents/{year}/{month}/
If regex
.*invoice.*
→ Finance/Invoices/
If extension
zip · rar · 7z · tar.gz
→ Downloads/Archives/
If extension
exe · msi · AppImage
→ Downloads/Installers/
Catch-all
everything else
→ Downloads/Other/

Offline & private

Your filenames never leave the machine.

  • ∅

    100% offline

    Zero cloud sync, zero filename uploads. Updates are optional and check-driven — not a background phone-home.

  • ⊘

    No telemetry by default

    History and settings stay in a local SQLite database under your user profile. Portable builds leave no registry footprint.

  • ⌀

    System files ignored

    desktop.ini, Thumbs.db, .DS_Store, browser temp stubs, and zero-byte placeholders are left untouched.

Quiet strengths
