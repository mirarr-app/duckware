What it is

A maintained fork that still ships upgrades

Topgrade started as a single Rust binary that walks your environment, figures out which updaters you actually have installed, and invokes them in a sensible order. This project is the actively maintained fork of the original r-darwish/topgrade, keeping the “just run topgrade” workflow alive across Linux, macOS, and Windows.

$ topgrade that’s the whole happy path

Why it holds up

Three reasons this stays on the PATH

DETECT

Auto-discovers your stack

No curated checklist of managers to remember. Topgrade probes what’s present — from apt and Homebrew to cargo, pip, winget, Nix, Flatpak, and more — then runs only the steps that apply.

CONFIG

TOML control without friction

Drop a config under your platform’s config dir, merge snippets from topgrade.d/, and hook custom commands before, during, or after the built-in steps. First run scaffolds a starter file for you.

REMOTE

SSH upgrade fan-out

List hosts under remote_topgrades and Topgrade will SSH in and run itself there before touching the local machine — with an optional host limit flag when you only want a subset.

Install

Pick a channel, get the binary

Official builds cover cargo, Homebrew, Winget, PyPI, deb-get, AUR, and Fedora Copr. Community packs land on Chocolatey, Scoop, MacPorts, Pacstall, Nix, Alpine, and Void.

Config & ops

Where settings live, how remotes join

Config paths

Priority wins: the first file found is used even if invalid. On first launch with nothing present, Topgrade writes a starter config automatically.

1. CONFIG_DIR/topgrade.toml
2. CONFIG_DIR/topgrade/topgrade.toml
+ merge CONFIG_DIR/topgrade.d/*
Windows: %APPDATA% · Unix: ~/.config

Remote execution

Declare hostnames in config; Topgrade SSHes out and runs itself remotely first. Narrow the blast radius with --remote-host-limit when you don’t want the full fleet.

example: remote_topgrades = ["nas", "devbox"]

Breaking changes bump the major version and land in release notes. MSRV lives in Cargo.toml under rust-version and only moves on majors. Prefer an example config as your starting template.
