Why it sticks

One phrase. Two machines. Encrypted bytes.

Most CLI transfer tools force a tradeoff: local-only, or insecure, or locked to one OS. croc refuses that bargain — relayed connectivity, PAKE-based encryption, resume, and real cross-platform installs in a single binary workflow.

01

Any two hosts

Public relay (or yours) stitches peers together. No port-forwarding, no shared LAN required.

02

PAKE encryption

The code phrase drives password-authenticated key agreement. The relay never sees plaintext.

03

Resumable drops

Interrupted transfers pick up where they left off — folders, multi-file batches, and pipes included.

04

Everywhere

Windows, Linux, macOS, Termux, FreeBSD, Nix, Docker — same mental model on every box.

The ritual

Speak the phrase, move the payload

The UX is deliberately tiny. One side invents (or customizes) a code phrase; the other types it. That phrase is the entire handshake.

Step 01

Send

croc send ./folder prints a short phrase and waits. Clipboard copy is on by default.

Step 02

Agree

PAKE turns the shared phrase into session keys. Relay traffic stays opaque to the middleman.

Step 03

Receive

On the other machine: croc <phrase> (or CROC_SECRET=… croc on Linux/macOS).

Audit notes

What makes this worth keeping installed

Beyond the cute crocodile branding, croc earns its place by covering edges other “just scp it” advice quietly ignore — NAT traversal, Tor-friendly proxies, and self-hosted relays when you want the middle box under your own roof.

↔

Relay without surrender

Default public relay for convenience; croc relay or Docker when you want private infrastructure and a password gate.

⌘

Pipes, text, QR

stdin/stdout piping, --text snippets, and QR for mobile handoff keep it useful outside “send this ZIP.”

⛨

Secret hygiene

Linux/macOS can pass the phrase via CROC_SECRET so process listings don’t leak the key (CVE-2023-43621 class issue).

⬡

Network realism

IPv6-first with IPv4 fallback, SOCKS5/Tor paths, curve and hash knobs for operators who care about the dials.
