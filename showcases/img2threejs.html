This instead

A TypeScript THREE.Group factory — reviewable, versioned, and wired with pivots, sockets, and colliders for motion.

Staged sculpt

Eight locked passes. Vision is the only approver.

Scripts gate each stage. The agent looks at one reference-vs-render sheet and chooses continue, refine-spec, refine-code, request-input, or stop — nothing else.

Token discipline

Scripts enforce. The model only judges.

Mechanical work — validation, gating, sheet packaging, pass state — stays in pure Python 3.10+ stdlib. Model context is reserved for visual judgment and code.

What you ship

Spec, factory, and evidence — not a blob.

Install as a Claude Code / Codex / OpenCode skill, point at an object image, and walk the gated forge until the side-by-side sheet clears threshold.

ObjectSculptSpec

Full component tree, materials, repetition systems, sockets, and a recorded review history for every pass.

TypeScript factory

createObjectModel() returns a Group with sculptRuntime for nodes, sockets, colliders, and destruction groups.

Comparison sheets

One packaged reference-beside-render image per pass — the only vision input the agent needs to score and decide.

Honesty about limits

A single image cannot reveal hidden sides or guarantee exact geometry. The skill says when output is approximate or stylized, mirrors unseen faces instead of faking confidence, and treats “cannot reach requested fidelity” as a valid result. Strong on hard-surface objects; characters stay stylized reconstructions.

Get the skill

Clone into your skills directory and invoke.

Pure Python stdlib under the hood — nothing to pip-install. Agent-agnostic vision wherever the host provides it.
