portless

Stable, named local URLs for dev servers — built for humans and coding agents alike.

Install via npm View Source

The Core Idea

package.json
// Before
"dev": "next dev" // http://localhost:3000

// After
"dev": "portless run next dev" // https://myapp.localhost

What is portless?

Local development is littered with ephemeral port numbers — localhost:3000, localhost:5173, localhost:4567. They shift between projects, collide across monorepos, and mean nothing to a browser bookmark or an AI agent trying to open your app.

portless from Vercel Labs sits in front of your dev server as a lightweight reverse proxy. It assigns each app a memorable hostname like https://myapp.localhost, handles HTTPS with HTTP/2 out of the box, and injects a PORTLESS_URL environment variable so both humans and automated tooling always know where to look.

Architectural Highlights & Strengths

  • HTTPS Proxy with Zero Ceremony

    On first run, portless generates a local certificate authority, trusts it in your OS keychain, and binds port 443. Browsers get clean HTTPS with HTTP/2 multiplexing — no certificate warnings, no manual mkcert setup. The proxy auto-starts when you launch an app and remembers its configuration across reboots.

  • Framework-Aware Port Injection

    Most frameworks respect the PORT environment variable automatically. For those that ignore it — Vite, Astro, Angular, Expo — portless detects the toolchain and injects the correct --port and --host flags. You run one command; the plumbing happens invisibly.

  • Monorepo, Worktree, and Sharing Modes

    A single portless.json at the repo root can route every workspace package. Git worktrees get automatic subdomain prefixes so parallel branches never collide. Optional Tailscale, Funnel, and ngrok integrations expose the same stable URLs to teammates or the public internet without reconfiguring your app.

Getting Started

Install globally, name your app, and run your existing dev command through the proxy:

terminal
npm install -g portless
portless myapp next dev
→ https://myapp.localhost

Monorepo Routing

For multi-package repos, a root config maps each workspace to its own subdomain. Run from the repo root to start everything, or cd into a single package:

portless.json
{
  "apps": {
    "apps/web": { "name": "myapp" },
    "apps/api": { "name": "api.myapp" }
  }
}

# https://myapp.localhost + https://api.myapp.localhost
