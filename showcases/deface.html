Audit note

What Deface actually does

Publishing field recordings, dashcam clips, or research video often means redacting bystanders by hand — dragging blur boxes frame by frame and hoping they stay aligned as people move. That workflow does not scale when you have folders of media or privacy rules that demand consistent coverage.

Deface is a command-line tool that runs a CenterFace detector on every video frame or still image, then paints an anonymization filter over each hit. Point it at a file, a directory, or cam for a live webcam demo. By default it writes an _anonymized sibling and drops audio unless you ask to keep it — a deliberate privacy-first posture for footage that should not leak identifiable voices either.

Detection pipeline

Find the face. Mask the region. Write the frame.

Each frame is scored independently with CenterFace (trained on WIDER FACE), so poses, scales, and partial occlusions still get a bounding box. Those boxes become elliptical or rectangular masks for the filter you choose.

Ingest

Pass a path, a shell glob, or a whole directory. Photos and videos share the same entry point.

Detect

CenterFace scores every candidate. Tune --thresh when you see false positives or missed faces.

Mask

Expand coverage with --mask-scale, switch to boxes, or downscale inference with --scale on high-res media.

Replace

Apply blur, solid black, mosaic, a custom image, or leave detections unmarked for debugging.

Anonymization modes

Blur, box, or mosaic — same detection, different redaction language

Defaults favor a strong Gaussian blur inside an ellipse. When you need harder redaction or a documentary look, swap the filter without changing the rest of the pipeline.

Blur

Strong Gaussian blur over elliptical face regions — the default that keeps scenes readable while identities dissolve.

--replacewith blur

Solid boxes

Opaque black rectangles for zero-detail redaction. Pair with --boxes when ellipses are not enough.

--boxes --replacewith solid

Mosaic

Pixel-block faces with a tunable tile width — familiar broadcast-style anonymization that still signals “someone was here.”

--replacewith mosaic

Batch CLI & live preview

One command for a folder. One shortcut for a webcam.

Deface is built for shell workflows — no GUI roadmap, just paths, flags, and ffmpeg under the hood for encoding.

shell 
# install
python3 -m pip install deface

# single video → vid1_anonymized.mp4
deface myvideos/vid1.mp4

# batch an entire directory
deface ./footage/

# mosaic + keep audio
deface clip.mp4 --replacewith mosaic -k

# live webcam demo
deface cam
Live capture

Webcam before you commit the batch

Run deface cam (a shortcut for preview on <video0>) to watch detection and filters on a live feed. Dial in threshold and mask scale on a short preview, then point the same flags at your directory of exports.

Need confidence overlays while tuning? --draw-scores prints faceness scores above each hit so you can raise or lower --thresh with evidence instead of guesswork.

Why it holds up

Practical privacy tooling, not a research toy

Frame-honest detection

CenterFace on WIDER FACE handles crowded scenes and awkward angles well enough for automated redaction — not perfect, but tunable when it misses or overfires.

Filters that match policy

Blur for soft redaction, solid for hard blocks, mosaic for broadcast habit, or a custom replacement image when you need a branded cover.

Shell-native scale

Glob expansion, directory inputs, optional preview, and ffmpeg encoding knobs keep anonymization inside the same scripts that already move your media.

Optional ONNX Runtime acceleration

Install onnxruntime-gpu, DirectML, or OpenVINO when you need faster inference. With a capable backend present, Deface prefers ONNX Runtime automatically and can target the fastest execution provider available.

CUDA DirectML OpenVINO --backend onnxrt
