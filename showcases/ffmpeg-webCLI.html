What is ffmpeg-webCLI?

ffmpeg-webCLI is a browser-native media editing tool that executes the compiled `ffmpeg.wasm` binary inside web background workers. It brings the full, scriptable power of command-line ffmpeg directly to standard browsers, letting users trim, convert, filter, crop, and chain operations on media files without servers.

By running completely client-side, the app offers a privacy-centric alternative to cloud video convertors. The files never leave the machine. There is no buffering queue, subscription wall, or file size upload limit except the local hardware threshold.

Key Strengths & Audit Highlights

  • Privacy-First Local Processing

    Cloud conversion services are subject to data breaches, surveillance, and tracking. By running the WebAssembly core locally on your device, your sensitive clips, audios, and screen captures remain confidential and secure.

  • Zero Re-Encoding Quality Loss (Stack Mode)

    Standard web utilities re-encode your video on every single change. ffmpeg-webCLI features an "Operation Chaining" stack mode that aggregates crop, adjust, resize, and grayscale filters into a single complex filter string, running a single encoding pass to avoid generation loss.

  • Offline-First PWA Support

    Bundled with a robust service worker that caches the 31 MB WebAssembly core, the tool works completely offline. It also implements the Screen Wake Lock API to prevent devices from sleeping or locking during lengthy video processing queues.

Raw FFmpeg Recipe Example

In addition to pre-set sliders, the dashboard features a direct command line. You can compose and execute custom argument arrays directly against the file system. For example, chaining a crop with brightness adjustment:

ffmpeg complex command
# Chain crop, adjust brightness/contrast, and output H.264 mp4
ffmpeg -i input.mp4 -vf "crop=1280:720:0:0,eq=brightness=0:contrast=1:saturation=0" -c:v libx264 -preset fast -c:a aac output.mp4
