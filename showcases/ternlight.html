Showcase #17 · Audit

Semantic search that never leaves the CPU

Hosted embedding APIs turn every keystroke into a network hop — latency, quotas, and a quiet leak of user text to someone else’s GPU. For browser extensions, offline note apps, edge workers, and static sites, that bargain is often unnecessary. You only need nearest-neighbor meaning, not a remote model farm.

Ternlight collapses that stack into one WebAssembly file: a BitNet-style ternary embedding model distilled from all-MiniLM-L6, a BERT tokenizer, and a hand-written Rust SIMD engine. Import @ternlight/mini (~5 MB, ~2.5 ms) or @ternlight/base (~7 MB, ~5 ms), call embed / similar, and search runs entirely on-device — Node, browsers, Workers, Deno, Bun.

Single artifact

One WASM bundle. Zero runtime fetch.

Model weights, tokenizer vocab, and inference kernels ship together. No postinstall download, no model registry ping, no sidecar microservice — the package is the engine.

Model ternary · int4
Tokenizer BERT · 128 tok
Engine Rust → WASM SIMD

Wire it once, run anywhere

Ternary weights (−1 / 0 / +1) turn matmul into adds and subtracts that ride CPU vector lanes. Quality holds because the student trains as a ternary model from the start — not a clumsy post-hoc quantize.

Two npm tiers share the same API. Pick by size and quality budget; both emit 384-dim L2-normalized vectors ready for cosine search.

@ternlight/mini 5.0 MB · ~2.5 ms
@ternlight/base 7.2 MB · ~5.1 ms

Inside the light

Tokenizer, model, SIMD — stacked, not scattered

Three pieces usually live in three repos and three deploy steps. Ternlight fuses them so “install the package” is the whole ops story.

01 — Tokenizer

BERT vocab in-bundle

Text is tokenized locally up to 128 tokens (~95 words). No remote tokenizer service, no surprise vocab downloads at first embed.

02 — Model

Ternary distilled MiniLM

Distilled from all-MiniLM-L6 with BitNet b1.58-style QAT. Base tracks the teacher at 0.844 Spearman; Mini stays close at a smaller footprint.

03 — SIMD engine

Rust kernels → WASM

Hand-written inference mirrors BitLinear’s forward pass byte-for-byte, compiled to WASM SIMD so CPU vector units do the real work.

Millisecond CPU inference

Faster than a round-trip can start

At a few milliseconds per embed on a laptop CPU, search-as-you-type is not aspirational — it is the default. The live demo ranks ~2,000 documents entirely in the browser; results appear before a cloud API would finish DNS.

Stay on device

The constellation never phones home

Queries and documents stay inside the process that loaded the WASM. That is not a privacy policy — it is an architecture.

Local only

Embeddings computed here. Vectors stay here.

  • No data-handling agreements User text never crosses a network boundary for inference. Ideal for notes, mail, support tickets, and regulated corpora.
  • Offline-first by default Extensions, Obsidian plugins, desktop shells, and static sites keep working without an embedding endpoint.
  • Edge without an inference sidecar Cloudflare Workers, Vercel Edge, Deno Deploy — embed next to the request handler; no GPU service to operate.
