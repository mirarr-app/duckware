Ternlight

Semantic embeddings in a 5–7 MB WASM bundle — search on CPU, no API calls, no GPU.

View Source Live Demo

Project Banner

What is Ternlight?

Most semantic search stacks depend on hosted embedding APIs — network latency on every keystroke, usage limits, and data leaving the device. For browser extensions, offline note apps, edge workers, or static sites, that trade-off is hard to justify when all you need is "find the closest matching sentence."

ternlight packs a distilled embedding model, BERT tokenizer, and SIMD inference engine into a single WebAssembly bundle under 7 MB. Embed text on CPU in a few milliseconds, run cosine similarity or nearest-neighbor search locally, and ship two npm tiers — @ternlight/mini for speed and @ternlight/base for quality.

Architectural Highlights & Strengths

  • Ternary Weights, Tiny Footprint

    Distilled from all-MiniLM-L6 with BitNet-style quantization-aware training. Every weight is −1, 0, or +1, so inference reduces to adds and subtracts on SIMD vector lanes. The result is a 5–7 MB gzipped WASM file that still tracks the teacher model closely on retrieval benchmarks.

  • One Bundle, Zero Runtime Fetch

    Model, tokenizer, and engine ship together — no postinstall download, no separate model registry, no inference microservice. Import from Node, browsers, Cloudflare Workers, Vercel Edge, Deno, or Bun and the package routes to the right loader automatically.

  • Search-as-You-Type on Device

    At ~2.5–5 ms per embed on CPU, results can appear before a network round-trip would even start. The live demo searches 2,000 documents entirely in the browser — a concrete proof that privacy-sensitive, offline-first, and edge-runtime apps can run real semantic search without calling OpenAI or Hugging Face.

Basic Usage

Three functions cover the common embedding workflow — embed a string, compare two vectors, or rank a corpus:

search.js
import { embed, cosineSim, similar } from "@ternlight/base"

cosineSim(
  embed("reset my password"),
  embed("I forgot my password")
) // → 0.88

similar("I want my money back", corpus, { topK: 2 })
