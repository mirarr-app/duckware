What is autobrr?

Private tracker users know the race: new releases appear, and the first seeders build ratio fastest. Tools like Radarr and Sonarr poll RSS feeds on a schedule — useful, but too slow to join the initial swarm on competitive indexers. Legacy automation scripts like autodl-irssi solved this with IRC, but they are brittle, hard to configure, and show their age.

autobrr modernizes that workflow into a single self-hosted application. It watches indexer IRC announce channels in real time, matches releases against your filters, and dispatches torrent files to qBittorrent, Deluge, Radarr, Sonarr, or a watch folder — all through a clean web UI with both dark and light themes.

Architectural Highlights & Strengths

  • Real-Time IRC Announce Monitoring

    Indexers broadcast new uploads to IRC the instant they hit the site. autobrr sits in those channels and reacts in milliseconds — not on the next RSS poll cycle. That speed difference is what separates early seeders from everyone still refreshing a feed.

  • Filter-Driven Automation

    Each filter is a rule set: match on release name, size, category, indexer, or regex patterns, then trigger an action. Actions can push to a download client, forward to Radarr or Sonarr for library import, or drop a torrent into a watch folder. Complex workflows that used to require multiple tools now live in one place.

  • Unified Torrent and Usenet Stack

    autobrr draws from the best ideas in trackarr, autodl-irssi, and flexget, then unifies torrent and Usenet automation under one roof. Whether you maintain ratio on private trackers or automate a Usenet pipeline, the same filter-and-action model applies across both ecosystems.

How a Filter Works

A typical setup monitors an indexer's IRC channel, matches incoming announces, and routes approved releases to your media stack:

filter flow
# IRC announce arrives from indexer
→ Filter: match "2160p" + category "Movies"
→ Action: send to Radarr + qBittorrent
✓ Release grabbed · seeding within seconds
