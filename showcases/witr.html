01 — Incident brief

Something is running. Prove why.

A mystery process chews CPU. A port you never opened is listening. A container showed up overnight. The usual response is a scavenger hunt across ps, lsof, ss, systemctl, and docker ps — then stitching the story together in your head.

Witr (Why Is This Running) closes that gap. Point it at a process name, PID, port, file lock, or container and it returns the causal chain: who spawned what, through which supervisor, and what context keeps it alive right now. One Go binary. One readable answer — or a live TUI when you need to roam the whole machine.

02 — Entry points

Four ways into the same chain

Everything collapses to a PID. Witr walks ancestry from there.

∋

Process tracing

Query by name or PID. Fuzzy match by default, exact with --exact. Output includes user, command line, start time, working directory, git repo, and the full supervisor chain — systemd → pm2 → node, not just a flat process list.

∷

Port forensics

Bind a port number and learn who owns the socket — then why that process exists. Ideal for “what is listening on 443?” moments without juggling ss and ps manually.

▣

Container origins

Search across Docker, Podman, nerdctl, Kubernetes/crictl, Incus, LXC/LXD, and FreeBSD jails. Match on name, image, command, or compose labels — then surface mounts, networks, and project metadata.

⌀

File lock holders

Resolve who holds a POSIX/FLOCK lock (or open fd interest) and walk that process back to its cause. Useful when a deploy stalls on a locked file and “lsof + guesswork” is the old playbook.

03 — Audit findings

Why this tool earns a slot

Finding A

Causality as the product

Most sysadmin CLIs dump state. Witr’s contract is narrower and sharper: explain the chain that justifies a running thing’s existence. Ports, containers, and locks are just different doors into the same ancestry walk — source, cwd, sockets, and supervisors included.

Finding B

Interactive TUI that stays useful

Launch with no args or -i for four live dashboards — processes (with ancestry side panel), ports, containers, and locks. Sort, filter, signal, and renice without leaving the terminal. Auto-refresh backs off under load; mouse support is there when you want it.

Finding C

One static binary, many escape hatches

Ships for Linux, macOS, Windows, and FreeBSD via brew, apt, conda, winget, npm, AUR, and a curl install script. Need scripts? --json. Need a quick ancestry line? --short / --tree. Need the kitchen sink? --verbose and --env.
