Witr

Why is this running? — trace processes, ports, and containers back to their cause in one command.

View Source Latest Release

Example Output

witr node

User : pm2
Command : node index.js
Started : 2 days ago

Why It Exists :
  systemd (pid 1) → pm2 (pid 5034) → node (pid 14233)

Source : pm2
Working Dir : /opt/apps/expense-manager
Sockets : 127.0.0.1:5001 (TCP | LISTENING)

What is Witr?

You notice a mystery process eating CPU, a port you do not remember opening, or a container that appeared overnight. The usual workflow is a scavenger hunt — ps for the PID, lsof for the socket, systemctl for the unit, docker ps for the container — and then mentally stitching the chain together yourself.

witr (Why Is This Running) answers the causality question directly. Query by process name, PID, port, file lock, or container and get a single human-readable report: what is running, how it started, what supervisor or shell chain spawned it, and what context it belongs to. An interactive TUI mode adds live dashboards for processes, ports, containers, and file locks.

Architectural Highlights & Strengths

  • Everything Maps to a Causal Chain

    Ports, services, containers, and commands all resolve to PIDs. Once identified, witr walks the ancestry tree back through supervisors, shells, and init systems — surfacing the source (pm2, systemd, Docker Compose, etc.), working directory, git repo, and open sockets in one pass.

  • Interactive TUI Dashboard

    Run without arguments to launch a four-tab terminal UI: live process list with ancestry side panel, open ports with owning processes, containers across Docker/Podman/K8s/Incus/LXC, and system-wide file locks. Sort, filter, send signals, and renice processes directly from the interface.

  • Single Static Binary, Everywhere

    Distributed as one Go binary for Linux, macOS, Windows, and FreeBSD. Available via Homebrew, apt, conda, winget, npm, AUR, and a curl install script. Output modes include short ancestry lines, tree views, JSON for scripting, and verbose environment dumps.

Basic Usage

Query by name, port, PID, or container — or launch the interactive dashboard with no arguments:

terminal
witr node
witr --port 5000 --short
witr --pid 143895 --tree
witr -i
# interactive TUI dashboard
