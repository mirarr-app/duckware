What is Screenshot.rocks?

Shipping a product update means producing visuals — README banners, landing page heroes, App Store previews, social posts. Most teams either open Figma and spend an hour framing a screenshot, or ship a flat PNG that looks unfinished. Screenshot.rocks closes that gap with a browser-based editor that wraps your capture in realistic device chrome.

Drop in a screenshot, pick a browser or phone frame, tweak the background and padding, and export a presentation-ready image. The entire app is open source under MIT, so you can use the hosted version at screenshot.rocks or run your own instance.

Architectural Highlights & Strengths

  • Instant Mockup Framing

    The editor supports browser window chrome and mobile device frames — iPhones, Android handsets, and more. Upload any PNG or JPEG, position it inside the frame, adjust background gradients or solid colors, and export at the resolution you need for docs, marketing, or store listings.

  • One-Click Browser Extensions

    Official extensions for Chrome, Firefox, and Edge let you capture the current tab and send it straight into the editor. No manual save-and-upload loop — screenshot, mockup, export in a single workflow from wherever you are browsing.

  • Self-Hostable and Hackable

    The full source is a standard JavaScript project you can clone and run locally with Yarn. Teams that want an internal mockup tool behind the firewall, or developers who want to extend frame types and export options, have the complete codebase to work from.

Self-Hosting

Run your own instance with a short setup — useful for air-gapped environments or custom branding:

bash
git clone https://github.com/daveearley/screenshot.rocks.git
cd screenshot.rocks
yarn install
yarn start
