Castor

Cast the real stream to your TV — not a laggy mirror of your screen.

Point it at a page or URL. Headless Chrome finds the video, ffmpeg shapes it for your panel, and DLNA delivers full-quality playback from your terminal.

View on GitHub Releases

Signal path

Four hops from URL to living room

Castor is a real-time media pipeline, not a screen sharer. Each stage stays on your LAN so the TV gets a clean, playable feed.

01

Extract

Headless Chrome + CDP watches network traffic, clicks to start playback, and snags the stream URL.

02

Resolve

ffprobe picks a height that fits your TV’s max_height — up to 4K if the panel can take it.

03

Transcode

Passthrough when the TV already accepts the codec; otherwise hardware H.264 or libx264. Whisper can burn subtitles.

04

Cast

A local replay server feeds DLNA MediaRenderer devices — Samsung, LG, Bravia, and friends.

Why bother

Mirroring cheats. Casting doesn’t.

Smart TVs refuse arbitrary web video. Screen mirrors crush resolution and add lag. Castor sends the stream the page already had.

Screen mirroring

  • Compressed desktop capture

  • Input lag on every frame
  • Tied to your laptop staying awake

What stands out

Built for people who already own the remote

CDP

Browser-grade extraction

Castor drives Chromium over the DevTools Protocol, then runs a short action pipeline — page click, largest iframe, click again — so playback actually starts on sites that allow automation.

TUI

Stay in the terminal

castor cast opens an interactive browser that searches titles via TMDB and casts without a GUI. Direct cast player / cast movie commands cover scripted flows.

SUBS

Whisper, burned in

Optional whisper.cpp transcription generates captions and ffmpeg burns them into the outgoing video — useful when your TV’s subtitle stack won’t cooperate with the source.

On the wire

The castor TUI

Browse titles, pick a device, cast — all without leaving the shell.

Quick start

Point, scan, cast

config.yaml + shell native · LAN required 
# Tell Castor which TV to use
device:
  name: "Living Room TV"
  type: dlna

# Cast a page or direct stream
$ castor cast player https://example.com/watch/…

# Or the interactive title browser
$ castor cast

Native binary preferred — it shares your TV’s network for SSDP discovery. Needs Chrome/Chromium, ffmpeg, and ffprobe on PATH. Homebrew cask on macOS; Docker with --network host on Linux LAN boxes only.

  1. Install via brew install --cask stupside/tap/castor or build with Go + cmake.
  2. Run castor scan, copy the exact device name into config.yaml.
  3. Cast a player URL — or add TMDB + your own authorized sources for the interactive browser.

Purpose & disclaimer

Castor hosts nothing and ships no catalog — it casts only pages, stream URLs, or sources you supply and are authorized to use. It does not decrypt or circumvent DRM. Lawful use is on you; do not use it to infringe copyright.
