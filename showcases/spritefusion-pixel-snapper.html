What is Pixel Snapper?

AI image models produce convincing pixel-art aesthetics, but the underlying structure is wrong — cells drift in size, the implicit grid resolution shifts across the canvas, and colors bleed outside a fixed palette. Manually redrawing or hand-aligning every sprite defeats the purpose of using AI in the first place.

Sprite Fusion Pixel Snapper is a Rust tool (with CLI, WASM, and web/desktop editions) that takes messy input and outputs grid-aligned pixel art. Pixels snap to a consistent cell size, colors quantize to a strict palette you can customize, and fine details like dithering are preserved where possible — making AI-generated sprites actually usable in tilemaps, isometric maps, and game asset pipelines.

Architectural Highlights & Strengths

  • Grid Detection and Palette Quantization

    Auto-detects pixel cell size or accepts a manual override with --pixel-size. Quantize to k colors or constrain output to an exact custom palette via comma-separated hex values. Batch-process entire directories in one command for sprite sheet cleanup workflows.

  • CLI, WASM, and Browser Editions

    Run locally with cargo run input.png output.png, embed the WASM module in your own pipeline via process_image(), or use the hosted tool at spritefusion.com. Same core algorithm across all targets — no dependency on a proprietary cloud API.

  • Built for Game Asset Pipelines

    Designed for AI-generated sprites that need grid alignment, procedural 2D art that does not naturally fit a tile grid, and textures that must scale cleanly without anti-aliased blur. Part of the broader Sprite Fusion ecosystem for true pixel art generation.

Basic Usage

Process a single image, batch a folder, or pin the grid size and palette explicitly:

terminal
cargo run input.png output.png 16
cargo run sprites/in sprites/out 16 --pixel-size 8
cargo run input.png output.png --palette "0d2b45,ffecd6,ff4d6d"
