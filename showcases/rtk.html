What is rtk?

Coding agents burn context on stdout they barely need: full test harness chatter, thousand-line diffs, permission columns from ls, and linter dumps that bury the one failing rule. You still pay for every token. rtk (Rust Token Killer) sits between the shell and the agent as a transparent proxy — the real command still runs; only the text that returns is filtered, grouped, truncated, and deduplicated for the model.

It ships as one Rust binary with no runtime deps, covers 100+ everyday commands, and adds under 10ms of overhead. Hook into Claude Code, Cursor, Codex, Gemini CLI, and other agents so git status becomes rtk git status automatically — the agent sees the compact form without changing how you prompt.

Command-aware filters

Not generic compression. Each tool — git, cargo test, pytest, rg, docker, aws — has a tailored reducer that keeps failures, hunks, and structure while dropping noise.

Hook, don’t rewrite prompts

rtk init installs rewrite hooks for major coding agents. Bash tool calls are rewritten before execution; adoption is transparent across sessions and subagents.

Native speed, measurable gain

Sub-10ms proxy cost vs tens of thousands of tokens saved per session. rtk gain graphs what you actually saved — proof, not vibes.

four reduction strategies

01
Smart filtering Strip comments, whitespace padding, and boilerplate the model already knows.
02
Grouping Aggregate files by directory, errors by rule, tests by failure reason.
03
Truncation Keep the signal window; cut redundancy that only pads context length.
04
Deduplication Collapse repeated log lines into counts so stack spam becomes one line.
