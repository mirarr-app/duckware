What is Tork?

GUI torrent clients bury search, download management, and provider configuration behind tabs and modal dialogs. For developers who live in the terminal — or anyone who just wants to grab an official Ubuntu ISO without wading through sketchy mirrors — a keyboard-driven TUI can be faster and calmer than opening a desktop app.

tork is a Go CLI that unifies torrent search, download queueing, and Linux ISO browsing in one terminal interface. Query public indexes and custom Torznab feeds, deduplicate releases, rank by swarm health, preview magnets, and manage active downloads — pause, verify, seed, move, or relink — without leaving the keyboard.

Architectural Highlights & Strengths

  • Search, Rank, and Download in One Flow

    Search across Knaben, YTS, Nyaa, and your own RSS or Torznab feeds. Duplicate releases are grouped and ranked by seeders, size, and source quality before you commit. Preview a magnet with one key, queue it with another, then manage the full lifecycle from the downloads screen.

  • Official Linux ISOs, No Mirror Hunting

    A dedicated ISO browser pulls current official images from Ubuntu, Debian, Fedora, Arch, NixOS, Proxmox, and more. Pick a distro, grab the latest release, and skip the usual round of verifying checksums on third-party mirror sites.

  • Local Health Checks and Swarm Compass

    Run tork doctor to validate config, disk space, and provider reachability — read-only by default, with optional engine and history recording. The swarm compass tracks source and download health over time locally in ~/.tork/health.json, never uploaded anywhere.

Install

Available via Homebrew, AUR, Nix, or direct Go install. Config lives in ~/.tork/; downloads default to ~/Downloads/tork.

install
brew tap melqtx/tap && brew install tork
yay -S tork
nix run github:melqtx/tork
go install github.com/melqtx/tork/cmd/tork@latest
