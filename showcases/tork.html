01 · audit

What is Tork?

Most torrent workflows split across a browser for finding magnets, a desktop client for downloading, and yet another tab for verifying an ISO checksum. Tork folds that path into one Go-powered TUI: type a query, compare live swarms, preview a magnet, and manage the queue — all from the same calm terminal surface.

Built by melqtx, it targets people who already live in a shell — developers grabbing distro images, operators who want SOCKS5/Tor-aware transfers, and anyone who’d rather hit one key than hunt sketchy mirrors. Config stays in ~/.tork/; downloads land in ~/Downloads/tork unless you point them elsewhere.

  • One keyboard, full lifecycle Search → rank → preview → queue → pause, verify, seed, move, or relink without leaving the TUI.
  • Official Linux ISOs built in Browse current images from Ubuntu, Debian, Fedora, Arch, NixOS, Proxmox, and more — no mirror scavenger hunt.
  • Local health, not telemetry tork doctor and the swarm compass keep provider and download health history on disk only.

02 · search indexes

You name it, the cat fetches it

Query Knaben, YTS, Nyaa, plus your own RSS or Torznab feeds. Paste a magnet, infohash, or .torrent path into the same home box — nothing downloads until you confirm.

Multi-source search

Cast a net across public indexes and private feeds in one pass. Duplicate releases get grouped so you’re not scrolling three copies of the same encode.

Quiet file preview

Open a magnet or torrent to inspect files before committing. Metadata is cached under ~/.tork/metainfo/ so reopen doesn’t depend on hunting peers again.

04 · download management

Keep the queue calm

From the downloads screen: pause, seed, fully verify completed data, move files, relink, copy paths or magnets, and remove — single-key bindings throughout.

Verify for real

Rehash completed pieces in-app. Direct downloads with a published SHA256 get mismatch quarantine as .corrupt before retry.

Proxy-aware

tork proxy tor routes searches and peers in strict SOCKS mode — DHT and direct fallbacks stay off so traffic doesn’t leak.

Autopilot (WIP)

Describe a haul — seasons, codecs, size caps — and let the cat propose picks with reasons before anything queues.

06 · tork doctor

When something feels off

Read-only by default: config, disk, state, cached metadata, and provider reachability. Opt into engine checks, proxy egress verification, or recorded health history — never uploaded.

✓ config ~/.tork/config.yaml · mode 0600
✓ disk ~/Downloads/tork · 84G free
✓ provider knaben · 210ms
✓ provider nyaa · 340ms
~ proxy check skipped · use --proxy-check
$ tork doctor --record

Press H in the TUI for source and swarm history, or enable a local daily canary check in config. The swarm compass shows whether your sources stay healthy over time — on your machine only.

07 · install

Bring the cat home

Homebrew, AUR, Nix, or Go 1.26+. Requires a terminal; thrives with a keyboard.

# macOS
brew tap melqtx/tap && brew install tork

# Arch
yay -S tork

# Nix / Go
nix run github:melqtx/tork
go install github.com/melqtx/tork/cmd/tork@latest
