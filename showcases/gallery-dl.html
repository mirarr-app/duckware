gallery-dl

One command to archive image galleries from hundreds of sites — with the config depth of a proper Unix tool.

Read Documentation View Source

Typical Usage

bash
gallery-dl "https://danbooru.donmai.us/posts?tags=bonocho"
./danbooru/bonocho/1234567.jpg
./danbooru/bonocho/1234568.jpg
# 847 files saved to disk

What is gallery-dl?

If yt-dlp is the standard for pulling video from the web, gallery-dl occupies the same niche for still images and comics. It is a Python command-line program that accepts a URL, identifies the hosting site, and downloads every image in a gallery or collection — paginated albums, manga chapters, artist portfolios, and tagged search results included.

The project maintains extractors for a vast and growing list of image hosts, from mainstream platforms to niche communities. It runs on Linux, macOS, and Windows, ships as a pip package or standalone executable, and rewards power users with granular configuration that rivals tools ten times its age.

Architectural Highlights & Strengths

  • Extractor-per-Site Architecture

    Each supported host gets a dedicated extractor module that understands its pagination, metadata, and media URLs. When a site changes its API, only that extractor needs updating — the core download engine, filename logic, and archive system stay untouched. The supported-sites list spans art communities, booru-style boards, manga readers, and social platforms.

  • Filename Templates and Archive Control

    Output paths are fully programmable via format strings — embed artist names, tags, chapter numbers, or timestamps directly into directory structures. Built-in archive databases skip already-downloaded files across runs, and chapter filters let you grab only the range you need (e.g. chapters 10–20 in French).

  • Flexible Authentication

    Private galleries and rate-limited sites are handled through username/password pairs, browser cookie import (--cookies-from-browser firefox), or OAuth flows for platforms like Flickr, DeviantArt, and Pixiv. Credentials live in a JSON config file so scheduled runs do not need interactive prompts.

Installation

Install from PyPI, grab a standalone binary, or use your system package manager — Homebrew, Scoop, Snap, and Nix all ship it:

bash
python -m pip install -U gallery-dl
# or: brew install gallery-dl
# or: scoop install gallery-dl

Configuration

Per-site settings merge from a JSON config file found in standard XDG paths. Override download directories, set default filename formats, and store credentials in one place:

config.json
{
  "extractor": {
    "base-directory": "~/Downloads/galleries",
    "archive": "~/Downloads/gdl-archive.sqlite3"
  }
}
