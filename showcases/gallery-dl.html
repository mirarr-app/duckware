01 · What it is

A site-aware archivist, not a dumb wget.

gallery-dl is a cross-platform Python CLI that downloads image galleries and collections from a long list of hosting sites—art boards, social timelines, manga readers, boorus, and more. You pass URLs; extractors negotiate pagination, metadata, and formats so you get a coherent local tree instead of a pile of anonymous blobs.

Configuration is deep JSON (YAML/TOML optional): base directories, per-site overrides, filters, and filename templates that can pull IDs, tags, artists, dates, and chapter numbers into the path. Cookies, browser cookie import, username/password, and OAuth cover private or logged-in content when a public scrape is not enough.

02 · Why it earns a shelf

Built for people who re-run the same collection for years.

The archive database remembers what you already have. Revisit a tag search or an artist page and gallery-dl skips duplicates instead of rewriting disks—critical when collections grow into tens of thousands of files. Optional yt-dlp/FFmpeg hooks extend the same workflow to video and Pixiv ugoira without abandoning the gallery model.

Extractor breadth

Hundreds of site-specific extractors share one invocation pattern: pass a URL (or force an extractor prefix) and let the tool walk galleries the way the site structures them.

Programmable layout

Filename and directory templates turn remote metadata into predictable local paths—artist / tags / date / id—so archives stay searchable without a separate renamer.

Auth that matches reality

When CAPTCHA or private feeds block a bare session, cookies-from-browser and OAuth flows unlock the same libraries you already browse signed-in.

03 · Config & download log

JSON on the left. Dedup on the right.

A typical setup mounts a config under ~/.config/gallery-dl/, enables the archive, and tunes extractors per host. Below is the feeling of a live run—not a transcript of a real scrape.

config.json excerpt 
// illustrative — not a dump of defaults
{
  "extractor": {
    "base-directory": "~/Archives",
    "archive": "~/Archives/.archive.db",
    "filename": "{category}_{id}.{extension}",
    "directory": ["{category}", "{search_tags}"]
  },
  "downloader": {
    "retries": 4,
    "rate": null
  }
}

04 · Get it

Install once. Revisit collections forever.

Stable builds ship on PyPI (pip install -U gallery-dl); Homebrew, Scoop, Chocolatey, Snap, Docker, and standalone executables cover the rest. Active development has moved to Codeberg, with the GitHub mirror still widely linked.
