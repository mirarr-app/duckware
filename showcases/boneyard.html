Boneyard

Pixel-perfect skeleton loading screens, automatically extracted from your real UI.

View Source NPM Package

Visual Identity

What is Boneyard?

Hand-tuned loading screens are a maintenance nightmare. Designers tweak layouts, and developers spend hours updating custom gray bars and shimmer offsets, leading to layout shifts when loaded content snaps into place. Boneyard resolves this by programmatically extracting skeletons directly from your production UI layout.

It functions by visiting your running application using a headless browser at build time, snapshotting coordinates and layout geometry of marked elements across custom viewports, and generating a portable registry file. When data loads, it transitions seamlessly without shifting a single pixel.

Architectural Highlights & Strengths

  • Decoupled Build-Time Extraction

    Boneyard’s Vite plugin and CLI instrument your dev environments transparently. The headless engine crawls designated components (e.g. Skeleton name="...") at multiple responsive breakpoints to construct the layout map. Your production client is left with zero measurement overhead.

  • Multi-Framework Native Adapters

    Whether you use React (with Suspense compatibility), Vue, Svelte 5, Angular, Preact, or React Native, Boneyard uses a unified .bones.json configuration. The structural schemas are entirely swappable across client ecosystems.

  • Native Accessibility & Dynamic Text Scaling

    For mobile applications built in React Native, Boneyard captures shapes at default font settings, but automatically recalculates bone scaling and positioning at runtime. Skeletons adjust seamlessly to users' customized OS text size layouts.

Integration Pattern

Integrating Boneyard into your application is straightforward. Below is an example of wrapping a query boundary using React:

React Component
// Wrap asynchronous UI structures with a Skeleton boundary
import { Skeleton } from 'boneyard-js/react'

function BlogPage() {
  const { data, isLoading } = useFetch('/api/post')
  return (
    <Skeleton name="blog-card" loading={isLoading}>
      {data && <BlogCard data={data} />}
    </Skeleton>
  )
}

CLI Build Workflow

Run the extraction tool to crawl layouts and generate the static schemas:

bash
# Captures skeleton screens from running UI and saves to registry
npx boneyard-js build --out ./src/bones
Captured 4 skeleton screens across 3 breakpoints (375px, 768px, 1280px).
