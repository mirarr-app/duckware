kage

A website’s shadow, with the JavaScript cut out.

Clone any site in headless Chrome, keep the finished page, delete every script, and browse the mirror with zero network.

opened · sanitize.diff

Render first. Then redacted.

Browser “Save As” keeps a thin client that still expects live scripts. kage waits for Chrome to finish the page a human would see, snapshots that DOM, then tears out <script>, handlers, and javascript: URLs before anything touches disk.

opened · _kage/state.json

A crawl that remembers where it stopped.

Breadth-first, robots-aware, sitemap-seeded. Each URL maps to one on-disk path, so resumes and refreshes stay idempotent. Ctrl-C saves the frontier; run again and it continues.

opened · out/

One folder. Or one file. Or one app.

Folders are fine for reading. For carrying a site around, pack into an open ZIM (Kiwix-compatible), a self-serving binary, or a double-click wrapper. Optional webview builds open a real OS window instead of another browser tab.

out/example.com.zim ZIM Deterministic archive — same mirror, same bytes. Opens in kage or Kiwix. out/example binary Archive glued to kage. Recipients need nothing installed; cross-pack with --base. out/example.app app Desktop shell with the mirror’s favicon — macOS .app, Linux AppImage-style, Windows GUI exe.
