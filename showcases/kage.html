The problem

“Save As” was never yours

Browser saves leave behind thin clients: blank screens, endless spinners, and pages that still call home to trackers. kage takes the opposite path — render first, then rip the JavaScript out.

Browser Save As

A fragile snapshot that still expects live scripts, CDNs, and analytics endpoints that may vanish tomorrow.

vs

kage shadow

The finished DOM a human would have seen — CSS, images, and fonts localized — with every script deleted. Open it offline.

How it works

Render, then sanitize

A polite breadth-first crawl respects robots.txt and sitemaps. Chrome settles each page; sanitize strips scripts and handlers; assets land on deterministic local paths.

01

Seed

Start from a URL; expand via links and sitemap within scope.

02

Render

Headless Chrome waits for the live page to settle.

03

Strip

Scripts, handlers, and javascript: URLs leave the DOM.

04

Localize

CSS, images, and fonts rewrite to on-disk paths.

Quick start

Two commands to own a site

Clone into $HOME/data/kage/<host>/, then serve locally. Pack later if you want a single file.

shell 
# mirror the site (Chrome/Chromium required)
kage clone paulgraham.com

# browse the shadow offline
kage serve $HOME/data/kage/paulgraham.com
# → http://127.0.0.1:8800

# optional: one shareable ZIM
kage pack paulgraham.com
kage open paulgraham.com.zim

Why it stands out

Built for archives that last

影

Finished DOM, not source HTML

kage drives real Chrome so SPAs and lazy loaders finish their work before the snapshot. What you keep is what a person saw — then every script is gone.

∅

Offline means offline

No trackers, no phone-home, no surprise network calls. Plain HTML you can hand to a friend, open from a USB stick, or reopen in 2050.

▣

Pack once, open anywhere

Collapse a mirror into a Kiwix-compatible ZIM, a self-serving binary, a double-click app, or — with the webview build — a native OS window.

Pack shapes

Folder, archive, or executable

A mirror is great on disk. For sharing, kage pack collapses it into the shape you need — open format preferred.

.ZIM

Open archive

Deterministic ZIM packs work with kage, kiwix-serve, and the wider Kiwix ecosystem — boats, classrooms, long flights.

BINARY

Self-serving exe

Append the archive onto kage itself. Recipients need nothing installed; cross-pack with a release --base.

APP

Double-click shell

macOS .app, Linux AppImage-style bundles, or Windows GUI-subsystem viewers — site as app, no terminal flash.
